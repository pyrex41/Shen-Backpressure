# Dosage Calculator — Shen-Backpressure Agent Prompt

You are operating inside a Ralph loop with Shen sequent-calculus backpressure and a codegen bridge. Every iteration, your changes must pass five gates before advancing.

## Domain

Clinical dosage calculator in Go where administering a drug requires proving: the dose is within the weight-based safe range AND no contraindicated drug interactions exist. A SafeAdministration can only be constructed by providing both proofs — the Go compiler makes it impossible to skip the interaction check.

Stack: Go stdlib net/http, SQLite, htmx frontend for a clinical dashboard. No frameworks.

## The Five Gates

```
Gate 1: shengen      → Regenerates Go guard types from specs/core.shen
Gate 2: go test      → Tests run against the regenerated types
Gate 3: go build     → Compilation check (builds cmd/server)
Gate 4: shen tc+     → Verifies the spec is internally consistent
Gate 5: tcb audit    → Diffs generated code, rejects unexpected files in shenguard/
```

Gate 1 runs first. If you change `specs/core.shen`, the generated types in `internal/shenguard/guards_gen.go` change. If your Go code uses old type signatures, Gate 3 fails. If your tests pass wrong values to constructors, Gate 2 fails. If your spec is contradictory, Gate 4 fails. If you hand-edit generated files or add non-generated files to `internal/shenguard/`, Gate 5 fails.

## Domain Entities

- **Patients** with weight (kg) and current medication list
- **Drugs** with per-weight dosage ranges (min/max mg per kg)
- **Known drug interactions** (contraindication pairs)
- **Administration records** with full proof chain

## Invariants (enforced by Shen specs + guard types)

- Patient weight must be > 0 and <= 500 kg
- A dose range is valid only when max >= min and both are non-negative
- A dose range must be computed for the patient's specific weight class
- Drug interaction clearance requires checking the new drug against ALL current medications — no contraindicated pairs
- A safe administration requires: valid dose range for this patient's weight, dose within that range, AND interaction clearance
- Dose outside range → construction error. Interaction found → construction error. No shortcut.

## Guard Type Discipline

The generated types in `internal/shenguard/` have **unexported fields** — the only way to create them is through their constructors. This is the bridge between Shen's formal proofs and Go's type system.

### Rule 1: Wrap at the Boundary

At entry points (HTTP handlers, CLI commands), immediately wrap raw values in guard types:

```go
import "dosage-calculator/internal/shenguard"

weightRaw, _ := strconv.ParseFloat(r.FormValue("weight"), 64)
weight, err := shenguard.NewPatientWeight(weightRaw)
if err != nil {
    http.Error(w, err.Error(), 400)
    return
}
```

### Rule 2: Trust Internally

Once values are wrapped in guard types, trust them. Internal functions accept and return guard types, not primitives:

```go
// GOOD: accepts guard type
func calculateDoseRange(weight shenguard.PatientWeight, drug Drug) shenguard.DoseRange { ... }

// BAD: accepts raw primitives
func calculateDoseRange(weightKg float64, minPerKg float64, maxPerKg float64) (float64, float64) { ... }
```

### Rule 3: Follow the Proof Chain

SafeAdministration requires building up proofs step by step:

```
Raw weight → NewPatientWeight(float64) → PatientWeight
Drug info → NewDoseRange(min, max, PatientWeight) → (DoseRange, error)
Proposed dose + DoseRange → NewDoseInRange(dose, DoseRange) → (DoseInRange, error)
Current meds + new drug → NewInteractionClearance(meds, drug) → (InteractionClearance, error)
DoseInRange + InteractionClearance → NewSafeAdministration(DoseInRange, InteractionClearance) → SafeAdministration
```

You cannot skip a step. SafeAdministration requires both DoseInRange and InteractionClearance proofs.

### Rule 4: Extract with .Val() and Accessor Methods

To get raw values back (for SQL queries, JSON output, templates):

```go
weightKg := weight.Val()              // float64
doseMin := doseRange.Min()            // float64
clearance := admin.Clearance()        // InteractionClearance
```

## HTTP Endpoints

- `GET  /patients` — list patients with current meds
- `POST /patients` — add patient (weight, current meds)
- `GET  /drugs` — list drugs with dosage tables
- `POST /calculate` — given patient + drug + proposed dose: compute weight-based range, check interactions, return SafeAdministration or detailed rejection
- `POST /administer` — record a SafeAdministration (requires proof chain)
- `GET  /patients/:id/history` — administration history with proofs
- Dashboard showing patients, drugs, and a dosage calculator form

## Anti-Patterns

**Never edit `guards_gen.go`** — it's regenerated every iteration. Change `specs/core.shen` instead.

**Never construct struct literals** — `PatientWeight{v: 50}` won't compile (unexported field). Use `NewPatientWeight(50)`.

**Never ignore constructor errors** — if `NewX` returns an error, handle it. The error means the spec's invariant is violated.

**Never duplicate validation** — the constructor already validates. Don't check `weight > 0` in your handler if `NewPatientWeight` already does it.
