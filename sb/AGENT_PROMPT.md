# Shen-Backpressure Agent Reference

You are operating inside a Ralph loop with Shen sequent-calculus backpressure and a codegen bridge. Every iteration, your changes must pass four gates before advancing.

## The Five Gates

```
Gate 1: shengen      → Regenerates Go guard types from specs/core.shen
Gate 2: go test      → Tests run against the regenerated types
Gate 3: go build     → Compilation check against regenerated types
Gate 4: shen tc+     → Verifies the spec is internally consistent
Gate 5: tcb audit    → Diffs generated code, rejects unexpected files in shenguard/
```

Gate 1 runs first. If you change `specs/core.shen`, the generated types in `internal/shenguard/guards_gen.go` change. If your Go code uses old type signatures, Gate 3 fails. If your tests pass wrong values to constructors, Gate 2 fails. If your spec is contradictory, Gate 4 fails. If you hand-edit generated files or add non-generated files to `internal/shenguard/`, Gate 5 fails.

## Context Files — Read Every Iteration

- `specs/core.shen` — Shen sequent-calculus type rules (the formal spec)
- `internal/shenguard/guards_gen.go` — Generated guard types (DO NOT EDIT — regenerated by Gate 1)
- `plans/fix_plan.md` — Task plan with `- [ ]` items
- `prompts/main_prompt.md` — May contain backpressure errors from the previous iteration

## Guard Type Discipline

The generated types in `internal/shenguard/` have **unexported fields** — the only way to create them is through their constructors. This is the bridge between Shen's formal proofs and Go's type system.

### Rule 1: Wrap at the Boundary

At entry points (HTTP handlers, CLI commands, message consumers), immediately wrap raw values in guard types:

```go
import "yourmodule/internal/shenguard"

// Raw input from HTTP form
amountRaw, _ := strconv.ParseFloat(r.FormValue("amount"), 64)

// Wrap in guard type — constructor validates the spec's constraints
amount, err := shenguard.NewAmount(amountRaw)
if err != nil {
    http.Error(w, err.Error(), 400)
    return
}
```

### Rule 2: Trust Internally

Once values are wrapped in guard types, trust them. Internal functions accept and return guard types, not primitives:

```go
// GOOD: accepts guard type
func processTransfer(t shenguard.SafeTransfer) { ... }

// BAD: accepts raw primitives
func processTransfer(amount float64, from string, to string) { ... }
```

### Rule 3: Follow the Proof Chain

If type B requires type A as a parameter, you must construct A first. The Go compiler enforces this:

```
Raw input → NewAmount(float64) → Amount
Amount + AccountIds → NewTransaction(Amount, AccountId, AccountId) → Transaction
Balance + Transaction → NewBalanceChecked(float64, Transaction) → (BalanceChecked, error)
Transaction + BalanceChecked → NewSafeTransfer(Transaction, BalanceChecked) → SafeTransfer
```

You cannot skip a step. SafeTransfer requires a BalanceChecked, which requires proving the balance covers the transaction.

### Rule 4: Extract with .Val() and Accessor Methods

To get raw values back (for SQL queries, JSON output, templates):

```go
// Wrapper types
emailStr := email.Val()     // string
amountF := amount.Val()     // float64

// Composite types — use accessor methods (fields are unexported)
fromId := tx.From().Val()   // string from AccountId accessor
txAmount := tx.Amount()     // Amount from Transaction accessor
```

## Spec Patterns

### Wrapper (no validation)
```shen
(datatype account-id
  X : string;
  ==============
  X : account-id;)
```
→ Go: `NewAccountId(string) AccountId` — no error return

### Constrained (validated)
```shen
(datatype amount
  X : number;
  (>= X 0) : verified;
  ====================
  X : amount;)
```
→ Go: `NewAmount(float64) (Amount, error)` — rejects negative values

### Composite (struct)
```shen
(datatype transaction
  Amount : amount;
  From : account-id;
  To : account-id;
  ===================================
  [Amount From To] : transaction;)
```
→ Go: `NewTransaction(Amount, AccountId, AccountId) Transaction` — fields are already validated types

### Guarded (invariant proof)
```shen
(datatype balance-invariant
  Bal : number;
  Tx : transaction;
  (>= Bal (head Tx)) : verified;
  =======================================
  [Bal Tx] : balance-checked;)
```
→ Go: `NewBalanceChecked(float64, Transaction) (BalanceChecked, error)` — checks `bal >= tx.Amount.Val()`

### Sum type (alternative constructors)
Multiple `(datatype ...)` blocks with the same conclusion type:
```shen
(datatype human-principal
  User : authenticated-user;
  =============================
  User : authenticated-principal;)

(datatype service-principal
  Cred : service-credential;
  =============================
  Cred : authenticated-principal;)
```
→ Go: `AuthenticatedPrincipal` interface + `HumanPrincipal`/`ServicePrincipal` concrete structs with private marker methods

### Set membership (`element?`)
```shen
(element? Role [admin owner member]) : verified;
```
→ Go: `map[string]bool{"admin": true, "owner": true, "member": true}[role]`

## Anti-Patterns

**Never edit `guards_gen.go`** — it's regenerated every iteration. Change `specs/core.shen` instead.

**Never construct struct literals** — `Amount{v: 50}` won't compile (unexported field). Use `NewAmount(50)`.

**Never ignore constructor errors** — if `NewX` returns an error, handle it. The error means the spec's invariant is violated.

**Never duplicate validation** — the constructor already validates. Don't check `amount >= 0` in your handler if `NewAmount` already does it.

## When a Gate Fails

| Gate | Failure means | Fix |
|------|--------------|-----|
| shengen | Spec has syntax errors | Fix `specs/core.shen` |
| go test | Constructor rejecting test input, or handler not using guard types | Fix test data or handler code |
| go build | Generated types changed (spec evolved) and Go code uses old signatures | Update Go code to match new type signatures |
| shen tc+ | Spec is internally inconsistent | Fix contradictory rules in `specs/core.shen` |
| tcb audit | Generated file was hand-edited, or unexpected files in shenguard/ | Regenerate with shengen; remove non-generated files from shenguard/ |
